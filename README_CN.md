# 📂 Agent 项目管理 Skill

**跨平台项目上下文隔离与状态管理系统**

> 🌐 [English](README.md) | [中文](README_CN.md)

> 已在 **nanobot**、**OpenClaw**、**Hermes Agent** 等多个平台实战验证，解决 AI Agent 无状态性与项目连续性之间的根本矛盾。

## 简介

AI Agent 是无状态的，但项目需要连续性。本 Skill 通过为每个项目创建独立的 `STATUS.md` 文件，实现跨会话的无缝上下文恢复，同时不污染全局记忆。

- **物理隔离**：每个项目独立文件，不共享状态
- **意图识别**：无固定关键词，从上下文推断操作
- **分级模板**：完整模板 (A) 用于复杂项目，轻量模板 (B) 用于单任务
- **写入安全**：读-合并-写模式，减少意外覆盖
- **Session 内周期检查**：一次激活，持续自动扫描新对话，防止"聊了就忘"
- **增量 checkpoint**：只读新增对话，控制 token 消耗

👉 详细设计文档：[DESIGN_CN.md](DESIGN_CN.md)

## 解决什么问题

1. **记忆爆炸**：所有项目进度挤在一个文件 → 1000+ 行，检索慢
2. **上下文污染**：项目 A 的进度泄漏到项目 B 的上下文
3. **断点丢失**："继续上次的项目" → Agent 不知道是哪个、做到哪了
4. **话题遗忘**：讨论了新话题但没建项目记录 → 第二天全忘
5. **过度工程**：小任务被迫使用完整模板

## 解决方案

为每个项目创建独立的 `STATUS.md`——Agent 的"外部硬盘"：

```
workspace/projects/
├── index.md              # 自动维护的项目索引（含一句话概述）
├── {project-name}/
│   ├── STATUS.md         # 进度、待办、关键决策
│   ├── docs/             # 方案、报告
│   └── src/              # 代码、产出物
└── _archive/             # 已完成/已废弃项目
```

## 核心特性

| 特性 | 说明 |
|------|------|
| **物理隔离** | 每个项目独立 STATUS.md，不共享状态 |
| **意图识别** | 无死板关键词，从上下文推断新建/恢复/暂停/切换 |
| **分级模板** | 完整模板 (A) 用于多阶段项目，轻量模板 (B) 用于单任务 |
| **写入安全** | 读-合并-写模式，减少意外覆盖 |
| **索引自愈** | 自动创建 index.md，自动补录缺失项目 |
| **记忆边界** | 项目进度在 STATUS.md，用户偏好在全局记忆，不交叉 |
| **Session 周期检查** 🆕 | 激活后每 60 分钟自动扫描新对话，捕获新话题 |
| **增量 Checkpoint** 🆕 | 只读上次检查后的新消息，最小化 token 消耗 |
| **心跳兜底** 🆕 | 当 session 检查未激活时，心跳轮询作为最后防线 |

## 量化效果

| 指标 | 使用前 | 使用后 |
|------|--------|--------|
| 全局记忆大小 | 1153 行 | 167 行（减少 85%） |
| 项目恢复时间 | 用户重新描述 (~2 分钟) | 读取 STATUS.md (<1 秒) |
| 上下文污染风险 | 高（所有项目混合） | 低（物理隔离） |
| 小任务开销 | 完整 5 模块模板 | 轻量 2-3 模块 |
| 话题遗漏 | 总是丢失 | 周期检查自动捕获 |

## 安装

### nanobot
将 `SKILL.md` 复制到 `~/.nanobot/workspace/skills/project-manager/SKILL.md`

### OpenClaw
将 `SKILL.md` 复制到 `~/.enhance-claw/instances/{实例名}/workspace/skills/project-manager/SKILL.md`

### Hermes Agent
将 `SKILL.md` 复制到 Agent 的 skills 目录

## 工作流程

1. **新建项目** → 扫描索引检查关联 → 确实新则创建 → 与用户确认
2. **恢复项目** → 模糊匹配 → 强制读取 STATUS.md → 摘要同步
3. **保存状态** → 持久化结论 → 更新索引状态
4. **切换项目** → 保存当前 → 恢复目标
5. **归档项目** → 移动到 `_archive/` → 保留历史
6. **周期检查** 🆕 → 自动扫描新对话 → 捕获新话题
