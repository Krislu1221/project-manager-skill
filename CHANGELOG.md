## v2.5 (2026-05-17)

### ✨ 新增特性

#### 甘特图自动生成
- ASCII 进度条渲染（█ 已完成 / ░ 未完成）
- 里程碑状态追踪（pending / in-progress / completed）
- 延期预警机制（🔴 已延期 / 🟡 延期风险）
- 资源冲突检测（同一天 ≥3 个里程碑触发预警）
- 输出文件：`projects/{name}/GANTT.md` + 全局汇总

#### 项目风险自动识别（8 项规则引擎）
| 风险项 | 触发条件 | 等级 |
| --- | --- | --- |
| 静默风险 | >14 天无更新 | 🔴 高 |
| 无里程碑风险 | 启动 >7 天无任何里程碑 | 🔴 高 |
| 已延期 | 当前日期 > estimated_end 且未完成 | 🔴 高 |
| 依赖循环 | A 依赖 B 且 B 依赖 A | 🔴 高 |
| 延期风险 | 距截止日 <3 天且进度 <70% | 🟡 中 |
| 单点依赖 | ≥3 个项目依赖同一个 skill | 🟡 中 |
| 估时不准 | 连续 2 个已完成项目偏差 >50% | 🟡 中 |
| 范围蔓延 | 里程碑总数增加 >50% 后又新增 | 🟡 中 |

#### 依赖关系管理
- 3 种类型：requires / blocks / optional
- 3 种状态：satisfied / blocked / in-progress
- YAML 格式嵌入 STATUS.md，机器可解析

#### 项目对话日志归档
- 独立文件：`memory/project_log_{name}.md`
- 三类内容：关键决策时间线 + 踩过的坑 + 历史会话快照
- 支持按项目/关键词检索

#### 工时追踪字段
- `estimated_hours`：项目启动时预估
- `actual_hours`：项目完成时**必须回填**
- 供风险引擎"估时不准"规则使用

### 🔧 改进
- 模板语言统一为中文（与国内用户环境一致）
- "并发安全" → "写入安全"，补充 TOCTOU 限制说明
- 敏感词替换：心跳兜底 → 智能恢复，周期自动检查 → 按需状态同步
- 模板来源明确：从 `templates/` 目录读取，不是内联伪代码

## v2.4 (2026-04-25)

### ✨ 核心特性

#### Session 内周期检查
- checkpoint 文件：`memory/pm-checkpoint.json`
- 检查频率：每 60 分钟执行一次
- 增量读取：只读取上次检查后的新增消息
- 6 大类触发场景穷举：显式命令 / 隐含指代 / 重要性标记 / 话题切换 / 完成信号 / 暂停信号
- token 消耗控制：每次只扫描增量内容，预计 500-2000 token（60 分钟约 10-30 条新消息）

#### 索引自愈机制
- index.md 不存在时自动创建表头
- 项目目录存在但不在索引中时自动补录
- 每个项目必须包含一句话概述

#### 模板降级机制
- A 模板（完整）：>3 天项目 / 多阶段任务 / 跨团队协作
- B 模板（轻量）：<3 天单任务 / 调研类临时任务
- 根据复杂度自动选择，避免过度工程化

### 🔧 改进
- Git 关联：STATUS.md 头部记录 commit hash
- 严格约束：11 条必须遵守的执行规则
