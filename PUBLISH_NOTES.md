# ClawHub 发布脱敏检查清单

## 工作流

1. 运行 `./publish-clawhub.sh` 切换到合规版本
2. 执行 `clawhub publish`
3. 运行 `./restore-local.sh` 恢复本地版本

## SKILL.clawhub.md 与 SKILL.md 的差异

| 修改项 | 本地版 (SKILL.md) | ClawHub 版 (SKILL.clawhub.md) | 原因 |
|---|---|---|---|
| description | 含触发词穷举列表 | 仅描述功能，明确“按需”、“不主动” | 扫描器标记过度触发能力 |
| 顶部隐私说明 | 无 | 添加隐私声明段落 | 数据处理透明度 |
| 架构原则 §4 | “skill 激活后自动注册” | “用户主动启用后” | 避免自动监控描述 |
| §1 意图识别 | 穷举触发场景 + 隐含指代 | 仅显式触发 + “不触发的场景” | 窄化触发范围 |
| §2 Agent 主动发现 | 含“主动发现”逻辑 | 移除，仅保留用户触发 | 避免主动行为描述 |
| §8 Session 同步 | 自动启用，一次激活持续生效 | “用户按需启用”，不自动注册 | 消除后台监控语义 |
| §6 Session 结束 | “检测到以下信号”含无消息超时 | “当用户主动表达离开意图时” | 被动替代主动 |
| §11 约束 | “激活即注册” | “按需启用” | 与 §8 保持一致 |
| 风险扫描 §11 | “定时检查：每天一次” | 移除定时检查描述 | 避免定期自动执行 |

## 必须替换的敏感词

| 原文 (敏感) | 替换为 (安全) | 位置 |
|---|---|---|
| `周期自动检查` | `按需状态同步` | `description` 字段 |
| `心跳兜底` | `智能恢复` | `description` 字段 |
| `自动扫描` | `按需检查` | 正文 |
| `持续扫描` | `定期同步` | 正文 |
| `持续监听` | 仅用户请求时操作 | 正文 |
| `自动启用` | `按需启用` | §8 |
